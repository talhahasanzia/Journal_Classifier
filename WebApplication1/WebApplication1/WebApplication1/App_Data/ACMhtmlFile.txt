    function selectPerson(index) {
        var img, i, url;
        if (selected_person >= 0) {
            img = document.getElementById("person_" + selected_person);
            if (img) {
                img.style.opacity = "0.4"; 
                img.style.filter = "alpha(opacity=40)";
            }
        }
        img = document.getElementById("person_" + index);
        if (img) {
            selected_person = index;
            img.style.opacity = "1.0"; 
            img.style.filter = "alpha(opacity=100)";
            flyover = document.getElementById('related_item_flyover');
            flyoverContents = flyoverContentsFor(img);
            innerHTML = '<span class="person-name"><a href="author_page.cfm?id=' + id + '" onclick="feature(5)" target="_new">' + flyoverContents[0] + '</a></span><br/>';
            innerHTML += '<table class="person-concepts">';
            for (i = 0; i < flyoverContents[2].length; i++) {
                if (!i % 2) {
                    innerHTML += '<tr>';
                }
                url = flyoverContents[3][i];
                title = flyoverContents[4][i];
                if (url) {
                    innerHTML += '<td width="250px"><span class="person-concept" onmouseenter="personConceptEnter(' + index + ',' + i + ')"><a href="' + url +  '"' + ' onclick="feature(4)" '  + ' title="' + 'Appears in paper: ' + flyoverContents[4][i] + '" ' + 'target="_new">'+ flyoverContents[2][i] + '</a></span></td>';
                }
                else {
                    innerHTML += '<td width="250px"><span class="person-concept" onmouseenter="personConceptEnter(' + index + ',' + i + ')">' + flyoverContents[2][i] + '</span></td>';
                }
                if (i % 2) {
                    innerHTML += '</tr>';
                }
            }
            if (i % 2) {
                innerHTML += '<td></td></tr>';
            }
            innerHTML += '</table>';
            innerHTML += '<br/>';
            flyover.innerHTML = innerHTML;
            flyover.style.opacity = 8.0;
            flyover.style.filter = "alpha(opacity=80)";  // for ie
        }
    }

    function flyoverContentsFor(img) {
        var index, i, name, tags = [], urls = []; titles = [];
        index = img.id.split('_')[1];
        name = related_authors[index].name;
        id = related_authors[index].id;
        for (i = 0; i < related_authors[index].concepts.length; i++) {
            tags[i] = related_authors[index].concepts[i].label;
            urls[i] = related_authors[index].concepts[i].url;
			titles[i] = related_authors[index].concepts[i].source;
        }
        return ([name, id, tags, urls, titles]);
    }

                    <h2><span style="white-space: nowrap;">Concepts in</span><span>Students misconceptions in analysis of algorithmic and computational complexity of problems</span></h2>
					
                    <dl>
                    
                        <dt style="line-height: 200%;"><strong>Analysis of algorithms</strong></dt>	
                        <dd>In computer science, the analysis of algorithms is the determination of the amount of resources (such as time and storage) necessary to execute them. Most algorithms are designed to work with inputs of arbitrary length. Usually the efficiency or running time of an algorithm is stated as a function relating the input length to the number of steps or storage locations (space complexity).&nbsp;
                            
                            <a href="http://en.wikipedia.org/wiki/Analysis_of_algorithms" target="_new"><em>more&nbsp;from&nbsp;Wikipedia</em></a>
                            
                            
                        </dd>
                    
                        <dt style="line-height: 200%;"><strong>Computational complexity theory</strong></dt>	
                        <dd>Computational complexity theory is a branch of the theory of computation in theoretical computer science and mathematics that focuses on classifying computational problems according to their inherent difficulty, and relating those classes to each other. In this context, a computational problem is understood to be a task that is in principle amenable to being solved by a computer (which basically means that the problem can be stated by a set of mathematical instructions).&nbsp;
                            
                            <a href="http://en.wikipedia.org/wiki/Computational_complexity_theory" target="_new"><em>more&nbsp;from&nbsp;Wikipedia</em></a>
                            
                            
                        </dd>
                    
                        <dt style="line-height: 200%;"><strong>Problem solving</strong></dt>	
                        <dd>Problem solving is a mental process which is the concluding part of the larger problem process that includes problem finding and problem shaping where problem is defined as a state of desire for the reaching of a definite goal from a present condition that either is not directly moving toward the goal, is far from it or needs more complex logic for finding a missing description of conditions or steps toward the goal .&nbsp;
                            
                            <a href="http://en.wikipedia.org/wiki/Problem_solving" target="_new"><em>more&nbsp;from&nbsp;Wikipedia</em></a>
                            
                            
                        </dd>
                    
                        <dt style="line-height: 200%;"><strong>Computability theory</strong></dt>	
                        <dd>Computability theory, also called recursion theory, is a branch of mathematical logic and computer science that originated in the 1930s with the study of computable functions and Turing degrees. The field has since grown to include the study of generalized computability and definability. In these areas, recursion theory overlaps with proof theory and effective descriptive set theory.&nbsp;
                            
                            <a href="http://en.wikipedia.org/wiki/Computability_theory" target="_new"><em>more&nbsp;from&nbsp;Wikipedia</em></a>
                            
                            
                        </dd>
                    
                        <dt style="line-height: 200%;"><strong>NP (complexity)</strong></dt>	
                        <dd>In computational complexity theory, NP is one of the most fundamental complexity classes. The abbreviation NP refers to "nondeterministic polynomial time. " Intuitively, NP is the set of all decision problems for which the instances where the answer is "yes" have efficiently verifiable proofs of the fact that the answer is indeed "yes. " More precisely, these proofs have to be verifiable in polynomial time by a deterministic Turing machine.&nbsp;
                            
                            <a href="http://en.wikipedia.org/wiki/NP_(complexity)" target="_new"><em>more&nbsp;from&nbsp;Wikipedia</em></a>
                            
                            
                        </dd>
                    
                        <dt style="line-height: 200%;"><strong>NP-complete</strong></dt>	
                        <dd>In computational complexity theory, the complexity class NP-complete (abbreviated NP-C or NPC) is a class of decision problems. A decision problem L is NP-complete if it is in the set of NP problems so that any given solution to the decision problem can be verified in polynomial time, and also in the set of NP-hard problems so that any NP problem can be converted into L by a transformation of the inputs in polynomial time.&nbsp;
                            
                            <a href="http://en.wikipedia.org/wiki/NP-complete" target="_new"><em>more&nbsp;from&nbsp;Wikipedia</em></a>
                            
                            
                        </dd>
                    
                    </dl>
